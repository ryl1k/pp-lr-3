## 🤖 Типи дроїдів

### ⚡ Assault Droid (Штурмовий)
- **Здоров'я**: 100 HP
- **Урон**: 25 DMG
- **Особливість**: 30% шанс подвійної атаки (збільшується при невдачах)
- **Стратегія**: Високий урон, агресивна тактика

### 🛡️ Tank Droid (Танк)
- **Здоров'я**: 150 HP
- **Урон**: 15 DMG
- **Броня**: 20 ARM (зменшується від пошкоджень)
- **Особливість**: Поглинання урону, висока витривалість
- **Стратегія**: Захисна тактика, довгі бої

### 🎯 Sniper Droid (Снайпер)
- **Здоров'я**: 50 HP
- **Урон**: 50 DMG
- **Особливість**: 30% шанс критичного удару (+20 урону)
- **Стратегія**: Висока точність, швидке знищення

### 💚 Healer Droid (Лікар)
- **Здоров'я**: 75 HP
- **Урон**: 10 DMG
- **Лікування**: 25 HP
- **Особливість**: 50% шанс лікування союзника або себе
- **Стратегія**: Підтримка команди, довгострокова гра

## 🎨 Візуальні функції

### 🌈 Кольорова схема

## 💾 Технічні особливості

### 🏛️ Принципи OOP
- **Наслідування**: Всі дроїди наслідують базовий клас `Droid`
- **Поліморфізм**: Метод `takeTurn()` реалізований по-різному для кожного типу
- **Інкапсуляція**: Приватні поля з публічними методами доступу
- **Абстракція**: Інтерфейс `Battle` для різних типів боїв

### 🔧 Архітектурні рішення
- **Factory Pattern**: `DroidFactory` для створення дроїдів
- **Strategy Pattern**: Різні стратегії бою через `Battle` інтерфейс
- **Observer Pattern**: Візуалізація подій через `BattleVisualizer`

### 📝 Система логування
- Автоматичне збереження у `logs/` директорію
- Уніквальні імена файлів з timestamp
- Структурований формат для easy parsing
- Підтримка replay з візуальними ефектами
